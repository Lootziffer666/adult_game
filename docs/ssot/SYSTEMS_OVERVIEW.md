# SYSTEMS_OVERVIEW

Only systems needed for the current system-validation phase are listed here.

## 1) Context / Timing System (in scope)
- Tracks when/where actions occur.
- Gates event availability and interpretation opportunities.

## 2) Hint / Memory / Objective Support (in scope)
- Lightweight player-facing support.
- Can be hint lines, short journal/reminder notes, and completed event tracking.
- Must avoid heavy quest-marker overload.

## 3) Hidden Relationship / Reaction State (in scope)
- Internal state for how entities/world respond.
- Placeholder roles are allowed; final cast identity is not required yet.

## 4) World Reaction Trigger System (in scope)
- Actions create consequences that alter future context/information.
- Includes immediate and delayed reaction patterns.

## 5) Bounded Object / Evidence Hooks (optional but compatible)
- Small, meaningful evidence/object interactions are allowed.
- Do not escalate into broad combinatorial inventory gameplay.

## 6) Optional Pressure Modifiers (deferred-by-default)
- Rumor/public/hygiene style systems are optional modifiers.
- Include only if needed for the current proof question.

## Explicitly out of scope for this phase
- Final narrative canonization.
- Locked macro-route story architecture.
- Heavy content expansion before system confidence.
