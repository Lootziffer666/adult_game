# CURRENT_PHASE_NOTE

## Why there is no fixed story yet
The project is currently validating system behavior, not committing to final narrative canon.
A fixed story too early would hide whether the interaction model itself works.

## Why placeholders are allowed
Placeholder roles/scenes let the team test reaction logic, timing, and inference clarity without prematurely freezing cast/lore.
This is a deliberate prototyping strategy, not missing design discipline.

## Why system validation comes first
If interpretation-driven progression fails at prototype level, content expansion will multiply weak foundations.
Validation-first reduces downstream rework.

## Why legacy slices are references, not canon
Legacy vertical slices are valuable examples of cadence, tone, and interaction rhythm.
But they were created across shifting phases and contain conflicting assumptions.
They should inform design, not define current truth by default.

## Why Ren'Py remains current
Ren'Py is still the active implementation path for this prototype.
Engine change is a contingency only if Ren'Py proves to be a real blocker for the validated design goals.
